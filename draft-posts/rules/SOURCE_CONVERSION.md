# ソース変換ルール (Source Conversion)

このファイルは、Note 記事や外部テキストを本ブログ用の原稿へ再構成するためのルールです。
恒久的な編集ルールは `EDITORIAL_BASELINE.md`、カテゴリ戦略は `CONTENT_STRATEGY.md`、travel-diary の完成形は `TRAVEL_DIARY_RULES.md`、運用仕様は `OPERATIONS.md` を参照してください。

## 1. 役割

- 元原稿をそのまま整形するのではなく、素材を抽出してブログ記事へ載せ替える。
- ソース固有のノイズ除去と、カテゴリに合わせた再構成を扱う。
- frontmatter や命名規則は持たない。

## 2. 共通方針

- 変換後の記事の主語は元プラットフォームではなく本ブログに置く。
- 残すのは表現そのものより、体験の温度、失敗、発見、判断材料。
- 客観情報は必要に応じて補足確認し、断定だけを整える。
- 変換後は必ずブログ側のルールへ寄せる。

## 3. Note から残すもの

- その場で受けた強い第一印象
- 戸惑い、失敗、予定のズレ
- 同行者とのやり取りや空気の変化
- 店や街に対する素直な感想
- 現地で少し役立つ小さな情報
- 当時の未熟さや、あとから見て面白い反応

## 4. Note から持ち込まないもの

- Note 的な挨拶
- 前回、次回、回番号に依存した導線
- `スキしてね` `フォローお願いします` のような呼びかけ
- 画像がある前提で成立している短文
- 肩書きの繰り返しやプロフィール前提の自己紹介
- ハッシュタグ列

## 5. Note 特有のノイズ除去

次のキーワードや構造は、まず削除候補として扱う。

- `すべて表示`
- `スキありがとうございます`
- `Noteで読む`
- `フォローする`
- `まだ見ていない方は`
- `続きは次回`
- Note カードや埋め込み
- 1 文ごとの空行が続く改行過多

## 6. 変換フロー

1. Note 特有の前置き、導線、結び、ハッシュタグ、画像だけの行を外す。
2. 印象の強い場面を 3 から 5 個選ぶ。
3. 記事カテゴリへ載せ替える。
4. 主観と客観を分けて整える。
5. ブログ用の導入と締めへ置き換える。

## 7. カテゴリ別の載せ替え

- `travel-diary`: 出来事を時系列に並べ直し、その場の反応が自然につながるように書き直す。詳細は `TRAVEL_DIARY_RULES.md` を正本とする。
- `tourism`: 体験談を補助へ下げ、読者の判断に効く情報を中心へ寄せる。
- `itinerary`: 行程、予算、疲労感を主軸に組み替える。

## 8. travel-diary 変換時の追加ルール

- Note の勢いは参考にしてよいが、ブログ側では落ち着いた `です・ます調` へ整える。
- 「今日はここに行きました」の羅列にしない。
- 回番号、日数見出し、次回予告は持ち込まない。
- 画像がなくても読める文章にする。
- 小さな実用情報は流れを壊さない範囲で短く残す。

## 9. OK / NG

### NG

- Note の文章を少し整えただけで、導線や回番号が残っている。
- 画像がないと意味が通らない。
- 文体だけ寄せた結果、元プラットフォームの連載記事のままになっている。

### OK

- 元原稿の素材は残っているが、読んだときに本ブログの記事として成立している。
- その日の印象や感情が自然に追える。
- 実用情報は補足に留まり、記事全体の主役は体験に置かれている。
