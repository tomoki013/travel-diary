# コンテンツ種別ガイドライン (Content Type Guidelines)

このファイルは、記事カテゴリごとの「目的」「文体」「何を優先して書くか」を定義します。
全記事共通の原則は `WRITING_GUIDELINES.md` を参照してください。

## 1. tourism

### 目的

- 読者の検索意図に対して、実用的で信頼できる情報をまっすぐ返す。
- 現地で役立つ判断材料を提供する。

### 優先順位

1. 正確さ
2. わかりやすさ
3. 実用性
4. おすすめ度の明示
5. 最小限の体験談

### 書き方

- 主観は抑えめにする。
- ただし、完全に無味無臭にはせず、必要なら「初回訪問ならおすすめです」「時間優先ならこの方法が楽です」のような判断補助は入れてよい。
- 読者が迷いやすいポイント（料金、チケット、アクセス、混雑、注意点）は明確に分ける。
- 体験談を入れる場合も、読者の意思決定に役立つ範囲に留める。

## 2. itinerary

### 目的

- 行程と予算感を通じて、旅の全体像をつかませる。
- 読者が「この旅程は自分に合うか」を判断できる状態にする。

### 主軸

- **最優先は予算感。**
- あわせて、移動効率、疲労感、無理の有無、失敗ポイントも扱う。

### 書き方

- 日程の流れがわかる構成にする。
- 費用は総額だけでなく、交通費・宿泊費・食費などの粒度でも把握できるようにする。
- 「安く見えるが移動がきつい」「節約できるが朝が早くてしんどい」のように、金額と負荷をセットで書く。
- 読者が再現しやすいように、移動順・宿の位置・節約できた理由も書く。

## 3. series

### 目的

- 「その人の旅を読みに来る」体験を作る。
- 情報だけでなく、感情、失敗、発見、疲れ、高揚感まで含めて読ませる。

### 基本方針

- 語り手は原則として固定ペルソナを使う。
- まれに別著者が存在しても、その場合はこの persona ルールを適用しない。
- 単なる出来事の羅列ではなく、「その日どう感じたか」「なぜ印象に残ったか」まで書く。

### シリーズ別の濃淡

- **travel-diary:** 生活感、移動、食、疲れ、失敗、ひとり旅の自由さを強めに出す。
- **sunset:** 感傷や余韻を少し強めにする。ただしポエム化は避ける。
- **architecture:** 知的好奇心を前に出し、建築の見え方・構造・印象の解像度を上げる。
- **travel-history / landscape / kyoto など他シリーズ:** 上記を参考に、題材に応じて感情と情報の比率を調整する。

### 書き方

- 感情ははっきり書いてよい。
- 失敗やダサさ、弱音も価値として扱う。
- ただし読者が置いていかれないよう、情景・状況・感情の順で伝える。
- 前後の回とのつながりを意識し、シリーズとして自然に読めるようにする。

## 4. one-off

### 目的

- 単発でも読ませる切り口を作りつつ、誠実に価値を届ける。
- 旅行以外のテーマにも対応できる構成を取る。

### 基本方針

- SEO で読まれることを意識してよい。
- ただし、煽りすぎ・不安商法・誇張タイトルは厳禁。
- ランキング、比較、問いかけ型の構成は使えるが、結論は誠実に書く。

### 書き方

- 検索意図に対して、冒頭で結論を示す。
- 比較軸や判断基準を明確にする。
- 単なる感想文ではなく、「なぜそう言えるか」を示す。
- 旅行以外の題材でも、読者の判断を助ける設計を優先する。
