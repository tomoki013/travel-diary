# SEO / AIO 運用ガイドライン

このドキュメントは、「ともきちの旅行日記」におけるSEO（検索エンジン最適化）およびAIO（AI検索・AIエージェント最適化）の運用方針をまとめたものです。

## 1. 記事作成の基本方針

実体験に基づく独自の情報を、人間とAIの両方が理解しやすい構造で提供します。

### 記事タイプ別の構造化レベル

- **構造化を強める記事:** 観光地紹介、旅行ガイド、モデルコース、旅程、アクセス、費用まとめ、比較、旅行準備、注意点・失敗談。
- **最低限の調整に留める記事:** エッセイ的な旅行記、日記、雑記、運営報告、思考メモ。

## 2. 推奨される記事構成（実用系記事）

観光情報やアクセス情報などの実用系記事では、以下の要素を可能な限り含めてください。

- **この記事の要点 (AIO Summary):** 冒頭に3〜5点の箇条書きで、記事の主要な結論を記載します。
- **基本情報テーブル:** 訪問時期、エリア、予算、所要時間などを表形式でまとめます。
- **FAQセクション:** 読者やAIが抱きやすい疑問（「一人旅でも大丈夫？」「予約は必要？」など）に回答します。
- **関連記事への内部リンク:** 地域やテーマが関連する他の記事へ、文脈に合わせてリンクを貼ります。

## 3. 実体験情報の明示

AI生成コンテンツと差別化するため、以下の情報を積極的に記載してください。

- 具体的な訪問年月。
- 実際に支払った金額や利用した交通手段。
- 実際に体験して感じた「良かった点」と「微妙だった点・注意点」。
- 失敗談や、現地で困ったこととその解決策。

## 4. 技術SEO / AIO

- **Canonical URL:** 重複コンテンツを避けるため、常に正規URL（`https://tomokichidiary.com/posts/...`）を参照します。
  - 静的ページの metadata は `src/lib/page-metadata.ts` の `createPageMetadata()` で生成し、ページ自身のパスを canonical に設定します。
  - **禁止事項:** ルートレイアウト（`src/app/layout.tsx`）に `alternates.canonical` を置かないこと。全ページへ継承され、サイト全体がトップページの重複として扱われる事故につながります（2026-06 に実際に発生し修正済み）。
- **llms.txt:** AIエージェント向けに、サイトの概要と主要ページを `/llms.txt` で提供します。
- **構造化データ (JSON-LD):** `BlogPosting` や `BreadcrumbList` を適切に出力し、検索結果でのリッチリザルト表示を支援します。
- **画像SEO:** すべての画像に、内容を自然に説明する `alt` 属性を付与します。

## 5. 内部リンクとハブページ

サイト全体を「地域別」「目的別」の小さな専門サイトの集合体として整理します。

- 各地域の「まとめ記事」や、地域別ページ（`/destination/[region]`）をハブとして活用します。
- 関連する記事同士を繋ぎ、読者がサイト内を回遊しやすくします。

## 7. 季節性コンテンツの更新リスト

旅行需要に合わせて、以下の時期に記事のメンテナンスを推奨します。

### 春（3月〜5月）に向けた準備

- ゴールデンウィークの旅行先ガイド。
- タイのソンクラーン（4月）や暑い時期の注意点。
- 春の京都・桜の名所。

### 夏（6月〜8月）に向けた準備

- 夏休みの海外旅行準備、LCC利用ガイド。
- 暑い東南アジアでの服装、水分補給、日焼け対策。
- ヨーロッパのベストシーズン記事。

### 秋（9月〜11月）に向けた準備

- シルバーウィークの旅行、弾丸一人旅。
- 秋の京都・紅葉ライトアップ情報。
- 涼しくなった東南アジアや中東（エジプト、インド）のベストシーズン入り。

### 冬（12月〜2月）に向けた準備

- 年末年始の海外旅行、空港混雑対策。
- 冬のヨーロッパ（クリスマスマーケットなど）の注意点。
- 学生の卒業旅行向けモデルコースや費用まとめ。
